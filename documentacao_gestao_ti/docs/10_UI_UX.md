# Diretrizes de UI/UX

## Princípios

- Interface simples;
- Poucos cliques;
- Informações importantes visíveis;
- Design responsivo;
- Consistência visual;
- Evitar excesso de campos;
- Evitar telas poluídas.

## Padrões de tela

Todas as telas de listagem devem possuir:

- Título;
- Botão de ação principal;
- Filtro/pesquisa;
- Tabela paginada;
- Ações por linha;
- Estado vazio;
- Indicador de carregamento.

## Padrões de formulário

Todos os formulários devem possuir:

- Campos agrupados por contexto;
- Validação clara;
- Botão salvar;
- Botão cancelar;
- Mensagem de sucesso;
- Mensagem de erro compreensível.

## Cores e ações

- Ação principal: destaque visual.
- Cancelar: neutro.
- Excluir: alerta.
- Status positivo: verde ou equivalente.
- Status crítico: vermelho ou equivalente.
- Status intermediário: amarelo/laranja ou equivalente.

## Dashboard

O dashboard deve priorizar informações úteis para decisão:

- Chamados abertos;
- Chamados atrasados;
- Ativos sem agente;
- Ativos sem atualização recente;
- Equipamentos fora de garantia;
- Softwares críticos;
- Alertas de inventário.
