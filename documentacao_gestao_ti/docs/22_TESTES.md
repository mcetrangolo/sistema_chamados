# Testes

## Objetivo

Garantir estabilidade mínima da plataforma durante a evolução.

## Tipos de testes

- Testes unitários;
- Testes de integração;
- Testes de API;
- Testes de permissão;
- Testes do agente;
- Testes manuais de interface.

## Áreas críticas

- Login;
- Permissões;
- Cadastro de ativos;
- Envio de dados pelo agente;
- Abertura de chamados;
- Alteração de status;
- Relatórios;
- Auditoria.

## Critério mínimo

Antes de concluir uma nova funcionalidade, validar:

- Se não quebra funcionalidades existentes;
- Se respeita permissões;
- Se possui tratamento de erro;
- Se atualiza documentação quando necessário.
