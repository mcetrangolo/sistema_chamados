# Guia para Uso de IA no Projeto

## Objetivo

Orientar agentes de IA, como Codex, Copilot, Claude Code ou similares, a contribuir com o projeto de forma consistente.

## Antes de implementar

A IA deve ler:

1. `docs/00_PROJETO.md`
2. `docs/01_VISAO_PRODUTO.md`
3. `docs/02_ARQUITETURA.md`
4. `docs/03_PRINCIPIOS.md`
5. `docs/05_ROADMAP.md`
6. Documento do módulo relacionado
7. `docs/18_PADROES_CODIGO.md`

## Regras

- Não implementar funcionalidades fora do escopo sem justificar.
- Não duplicar componentes existentes.
- Não alterar arquitetura sem registrar decisão.
- Não criar dependências desnecessárias.
- Não inserir dados sensíveis no código.
- Atualizar documentação quando alterar comportamento relevante.

## Entrega esperada

Ao implementar algo, a IA deve informar:

- Arquivos alterados;
- O que foi implementado;
- Como testar;
- Impacto no banco de dados;
- Documentação atualizada.
